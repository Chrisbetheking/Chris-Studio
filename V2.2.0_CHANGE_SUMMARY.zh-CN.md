# Chris Studio v2.2.0 改造摘要

v2.2.0 不再继续堆叠分散页面，本轮优先收口“构建成功但发布失败”和 Agent 长任务缺少可靠状态的问题。

## 本批已经落地

1. **发布链路重构**：移除最终发布阶段对 `softprops/action-gh-release` 的依赖，改为仓库内可审计的 GitHub CLI 脚本。
2. **可重入发布**：Release 已存在时更新元数据；每个安装资源独立重试并覆盖同名资产；上传后再读取远端资产列表核验。
3. **失败前置**：发布标签与五处版本元数据必须完全一致；Tauri dialog feature、兼容 bundle identifier 和资源清单均在构建/发布前检查。
4. **双架构资源核验**：Apple Silicon 与 Intel 的 DMG、APP ZIP、安装助手、SHA-256 和签名说明缺一不可。
5. **Agent 可靠运行内核**：加入状态机、检查点、最大循环次数、默认最多三轮修复、等待审批、执行收据、补丁备份记录、取消和硬超时信号。
6. **Provider 运行遥测**：统一 OpenAI-compatible、Anthropic、Gemini 等常见 Token 字段，归一化认证、限流、配额、超时、网络和服务端错误；价格不硬编码，只接受显式费率配置。
7. **Computer Use 安全守卫**：加入一次性审批票据、票据过期与防重复消费、紧急停止、硬超时、坐标边界检查和可视化标记数据。
8. **可回滚收据**：根据补丁备份逆序生成路径安全的回滚计划，去重同一路径的连续修改，并支持 JSON 执行收据。
9. **自动测试**：新增状态机、三轮修复、Provider 遥测、Computer Use 守卫、回滚计划、发布脚本重试与可重入场景验证。
10. **Actions 运行时升级**：checkout/setup-node 升级到 v6，artifact 上传/下载升级到 v7/v8，移除 Node 20 退役警告来源。
11. **身份与版本同步**：构建前将侧栏版本同步为 v2.2.0，并把残留的 TokenFence 身份回答统一改为“我是 Chris Studio，由 Chris 全程设计和建造”；同时根工作区、桌面包、桌面 UI、Cargo 与 Tauri 配置统一到 2.2.0；保留旧 bundle identifier，避免破坏既有 macOS 权限和凭证迁移。

## 本批没有虚报为完成的内容

`reliableRun.ts`、`rollbackPlan.ts`、`providerTelemetry.ts` 与 `sessionGuard.ts` 已进入严格编译和测试链路，是 v2.2.0 统一工作台的底层可靠性内核；它们尚未被包装成新的独立页面，也没有被虚报成已经完成全部 UI 接线。后续应直接接入现有聊天工作台，不再制造分散控制页。

## 固定安全边界

- 不向模型开放任意 Shell；
- 不允许静默写文件、推送或创建 Pull Request；
- 不允许无人值守 Computer Use；
- API Key、GitHub Token 和 MCP Token 不写入明文配置；
- 不通过关闭 Gatekeeper 解决安装问题。
