# v1.7.0 GitHub Actions 安装修复

本修复针对 macOS 工作流 #4 的 `Install desktop UI dependencies` 失败。

## 根因

上一份生成包中的 `apps/desktop/ui/package-lock.json` 含有仅限生成环境访问的内部 npm Artifactory 地址。GitHub Hosted Runner 无法访问该地址，因此依赖安装等待数分钟后退出，Apple Silicon 和 Intel 构建均未启动。

## 本次改动

- 将桌面 UI 锁文件中的 246 个内部下载地址改为 npm 官方公共 Registry。
- 工作流固定使用 `https://registry.npmjs.org/`。
- 增加私有 Registry 地址拦截检查，避免同类问题再次进入构建。
- 不再安装整个仓库依赖，只安装桌面 UI 和 Tauri CLI，减少时间与不相关依赖影响。
- `checkout` 与 `setup-node` 升级至 Node.js 24 运行时版本，清除旧 Node.js 20 警告。
- 保留 TypeScript、隐私测试、Token 测试、Rust `cargo check`、双架构 DMG 和 Release 发布。

## 上传与运行

1. 解压 ZIP。
2. Finder 按 `Command + Shift + .` 显示 `.github`。
3. 把解压目录内部所有内容一次拖入仓库根目录并覆盖。
4. 提交信息：`fix: use public npm registry for v1.7 macOS builds`
5. 新建一轮工作流，不要 Re-run #4。
6. 参数：`version=v1.7.0`、`create_release=true`、`make_latest=true`。
