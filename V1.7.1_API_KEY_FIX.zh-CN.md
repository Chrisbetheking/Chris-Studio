# TokenFence Studio v1.7.1：API Key 读取修复

## 问题

在 v1.7.0 中，API Key 已成功写入 macOS Keychain 后，前端会按安全设计清空内存中的 Key。
但连接测试和正式对话在调用 Rust 后端之前，又错误地把“内存中没有 Key”判定为“没有配置 Key”，
因此出现：

```text
No DeepSeek API key is stored in the operating-system credential store.
```

这条错误来自前端提前拦截，不代表用户输入的 Key 无效，也不代表 Keychain 一定写入失败。

## 修复

- 只有在“没有临时输入 Key”且“配置也没有凭证标记”时，前端才提示未配置。
- 已安全保存的 Provider 会继续调用原生 Rust 后端。
- Rust 后端从 macOS Keychain / Windows Credential Manager 读取对应 Profile 的 Key。
- 如果系统凭证库中确实没有该条目，再由原生后端返回准确错误。
- 新增回归检查，防止后续版本重新在 WebView 层阻断凭证读取。

## 验证步骤

1. 输入新的 DeepSeek API Key。
2. 点击“保存并设为当前”。
3. 点击“测试连接”。
4. 成功后状态应变为“已连接”。
5. 关闭并重新打开 App，再次测试，Key 不需要重新输入。

DeepSeek 当前官方接口仍使用 `https://api.deepseek.com/chat/completions`，模型可选择
`deepseek-v4-flash` 或 `deepseek-v4-pro`。
