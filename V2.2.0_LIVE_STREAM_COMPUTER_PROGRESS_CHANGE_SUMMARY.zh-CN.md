# Chris Studio v2.2.0：实时流式与 Computer Use 进度修复摘要

## 本轮根因

### 对话看起来不是流式

后台 SSE 实际持续接收增量，但新会话开始后，父级侧栏立即选中了新会话 ID。工作台随后尝试从本地历史重新加载该 ID，而待处理会话尚未写入历史，因此内存会话被重置为空，主页覆盖了正在生成的助手消息。完成后会话才写入历史，所以用户看到的是整段回答一次性出现。

### Computer Use 只打开 TextEdit

原流程依赖模型逐步规划。非视觉模型可能在打开应用后提前返回完成，同时 `open -a TextEdit` 会在部分 macOS 版本恢复文档选择器，而不是创建无标题文档。界面也缺少清晰的步骤进度。

## 修复内容

- 新会话开始流式请求时，先保存待处理会话。
- 当前内存会话 ID 与侧栏选中 ID 一致时，不再从历史重复加载并清空状态。
- 请求运行时禁止显示工作台空主页，确保增量消息可见。
- 对“打开 TextEdit + 输入文字”建立受约束目标契约：创建无标题文档 → 输入批准文字 → 验证 → 完成。
- TextEdit 由 AppleScript 直接创建无标题文档，不再使用可能恢复文件选择器的 `open -a`。
- 输入文字时优先直接写入 TextEdit 前台文档，并保留辅助功能按键回退。
- Computer Use 新增可见步骤列表、百分比、当前步骤与执行日志。
- 为整个 Computer Use 目标建立一个父级可靠执行记录。
- 应用打开动作纳入一次性审批票据和紧急停止保护。
- Computer Use 规划请求不再为每一轮模型规划额外创建 Provider 运行卡片。
- 新增流式会话状态与 Computer Use 目标契约回归测试。
