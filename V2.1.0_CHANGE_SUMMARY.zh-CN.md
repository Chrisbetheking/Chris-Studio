# Chris Studio v2.1.0 改造摘要

## 本次收口的问题

1. **身份错误**：底层 Provider 不再对外自称 DeepSeek/OpenAI。询问“你是谁”时由本地立即回答：Chris Studio，由 Chris 全程设计与构建。
2. **联系方式**：内置邮箱 `chriswangjob@163.com` 与 WeChat `easymoneysniperchris`，About 页面同步展示。
3. **发送等待感**：安全内容一次点击完成本地审查并发送，支持 `⌘/Ctrl + Enter`；等待模型时展示实时耗时与处理阶段。
4. **功能分散**：在同一个对话框中加入 Codex 风格本地命令：
   - `/project`
   - `/git status`
   - `/git diff`
   - `/check npm-test`
   - `/skills`
   - `/permissions`
   - `/screen`
   - `/type 文本`
   - `/click x y`
   - `/key cmd+s`
5. **macOS 权限列表没有 Chris Studio**：新增原生权限探针，先触发 Accessibility 与 Screen Recording 授权，再打开系统设置；授权后需完整退出并重启应用。
6. **Logo 与颜色**：替换为黑曜石、象牙白、紫罗兰和小面积暖金的新图标与品牌色，不再使用原青绿色方案。
7. **新工作流构建失败**：CI 现在明确安装 `apps/desktop/ui` 的独立依赖并检查 `@tauri-apps/api`、`exceljs`、`pdfjs-dist`、`mammoth`、`tesseract.js`；旧 `release.yml` 已退役为提示工作流，避免重复发布。

## 安全边界

- Computer Use 仍然逐项确认；
- `/click`、`/type`、`/key` 执行前仍弹出确认；
- 项目读取只允许用户选择的目录；
- Git、构建与测试仍使用白名单预设，不开放任意 Shell；
- 高风险敏感内容仍必须使用脱敏版本。
