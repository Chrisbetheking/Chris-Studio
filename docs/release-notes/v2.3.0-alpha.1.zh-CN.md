# Chris Studio v2.3.0-alpha.1

这是 v2.3 系列的首个测试版本。稳定使用仍建议保留 v2.2.0。

## 本轮新增

- 在同一个工作台中加入可见的多模型 Agent 协作：规划 → 执行 → 独立审查。
- Planner、Executor、Reviewer 可以分别指定已经连接的 Provider 配置，也可以沿用工作台或执行模型。
- Executor 结果继续使用真流式输出，不需要等待规划和审查全部结束后才一次性显示。
- 每个角色都生成独立状态与模型回执，完整计划和审查结论随会话持久化。
- Reviewer 可以仅记录问题，也可以触发一次有上限的修订；不会进入无限修复循环。
- 已明确配置的角色模型缺失、停用或没有凭据时会直接报错，不会静默切换到其他模型。
- 停止按钮会终止整个协作流程，包括规划、执行、审查和有限修订。
- Token 预算会显示并估算三次或最多四次模型调用。

## 安全边界

- 文件写入、命令、GitHub、MCP tools/call 和 Computer Use 仍要求明确审批。
- 模型不得在没有真实工具回执时宣称已执行本地或外部操作。
- v2.2.0 已验收通过的会话、流式输出、Computer Use 和可靠执行中心链路保持不变。

## 本次重点验收

1. 在 Agent 配置中启用“规划 → 执行 → 审查”。
2. 使用同一个 DeepSeek 配置承担三个角色，确认计划先出现、正文实时流式生成、审查回执最终保存。
3. 为 Reviewer 指定一个不存在或停用的配置，确认任务明确失败且没有静默回退。
4. 在执行阶段点击停止，确认后续 Reviewer 不会继续调用。
5. 重启应用后确认计划、角色状态和审查回执仍在会话中。
