# Chris Studio v2.2.0 第三批改造摘要

## Codex 式工作台

- 左侧边栏加入最近 8 条对话。
- 支持打开、重命名、删除和进入完整历史归档。
- 当前对话与工作台状态联动高亮。
- 历史页改为稳定的单列归档布局，修复重叠、溢出和空白操作按钮。
- 右侧请求检查器默认收起，普通聊天不再长期占据屏幕宽度。

## 语言与安全流程

- 顶部新增明确的 `中 / EN` 快捷切换。
- 请求内容不再在输入期间执行真实载荷扫描。
- 点击发送后固定请求快照，再进行本地安全检查和脱敏。
- 严重风险仍必须确认脱敏内容；普通安全请求直接进入模型。
- 设置页移除误导性的“实时扫描”开关，改为仅在发现风险时展开检查器。

## DeepSeek 真流式输出

- Tauri 后端新增 OpenAI-compatible SSE 流式读取。
- 支持 DeepSeek 正文增量和 `reasoning_content` 增量。
- 流式 HTTP 读取转移到阻塞工作线程，避免冻结 Tauri 命令线程。
- 增加立即停止、后端取消标记、流结束清理和长回答 8192 Token 输出上限。
- Provider 忽略流式参数时自动兼容普通 JSON。
- 可靠执行中心记录流式请求、检查点、错误分类和重连状态。
- 仅在首个可见增量之前重连，防止已有回答重复追加。

## 模型驱动 Computer Use

- 用户描述最终目标，模型每次只规划一个动作。
- 形成“规划 → 执行 → 观察 → 再规划”的有限循环。
- 单次会话最多 8 步，始终保留紧急停止。
- 点击、输入、按键、终端和系统设置仍需逐项确认。
- 模型动作严格限制在应用、按键和坐标白名单。
- 非视觉模型禁止猜测坐标；可用于测试打开已知应用、输入文本和快捷键流程。
- 原手动截图、点击、输入和按键工具移入开发者折叠区。

## 发布链恢复

- 恢复可重复执行、分资源重试和远端资源核验的 Release 脚本。
- checkout/setup-node 升级到 v6，upload-artifact 升级到 v7，download-artifact 升级到 v8。
- 新增发版输入和仓库版本一致性检查。
- Apple Silicon 与 Intel 产物仍独立构建和发布。
