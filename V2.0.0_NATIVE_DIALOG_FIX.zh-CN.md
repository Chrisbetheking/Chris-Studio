# Chris Studio v2.0.0 原生构建修复

## 真实错误

Rust 编译失败于两处目录选择器调用：

```rust
tauri::api::dialog::blocking::FileDialogBuilder::new().pick_folder()
```

Tauri 1.8.3 的 `tauri::api::dialog` 模块受基础 Cargo feature `dialog` 控制。仅有配置入口但未稳定保留该 feature 时，Rust 编译会报 `cannot find dialog in api`。

## 修复

```toml
tauri = { version = "=1.8.3", features = ["dialog", "dialog-open"] }
```

并在 `tauri.conf.json` 中显式启用：

```json
{
  "tauri": {
    "allowlist": {
      "dialog": {
        "open": true
      }
    }
  }
}
```

工作流新增配置断言和 `cargo check`，以后会在打包 DMG 前直接暴露原生 Rust 错误。
