# Chris Studio v2.2.0 第二批：可靠执行工作台

本补丁将 v2.2.0 第一批已经完成的可靠性底层能力接入 Chris Studio 当前统一工作台，使模型请求与已批准的 Computer Use 操作拥有可见、可停止、可追溯的执行状态。

## 本批完成

- 新增全局“可靠执行中心”，显示运行中、需处理和历史执行记录。
- 模型请求接入可靠执行适配器：
  - 最多 3 次自动修复重试（共最多 4 次请求）；
  - 仅对限流、超时、网络和服务端错误重试；
  - API Key、权限、参数等不可重试错误直接返回；
  - 显示 Provider、模型、尝试次数、检查点和归一化错误；
  - 总任务硬超时会真实终止等待，不会只改变界面状态。
- Computer Use 接入可靠执行适配器：
  - 保留现有工作台的显式确认边界；
  - 每个已批准动作生成一次性审批票据；
  - 截图、点击、输入、按键分别产生执行收据；
  - 点击操作记录批准坐标；
  - 五分钟硬超时；
  - 全局紧急停止可以中断尚未返回的本地操作。
- 执行状态持久化：
  - 最多保留 40 条执行收据；
  - 应用重启后，之前未结束的任务会标记为“被重启中断/已停止”，不会永久假装仍在运行；
  - 已完成、失败、停止和超时记录可统一清理。
- 中英文界面跟随当前应用语言，支持键盘 Esc 关闭和 reduced-motion。
- 将工作台和 Computer 页面现有调用安全替换为适配器，不改变原函数签名。
- 新增状态仓库、工作台接线和幂等同步防回归测试。

## 未改变的安全边界

- 不开放任意 Shell。
- 不允许无人值守的后台电脑控制。
- Computer Use 仍要求现有界面先获得用户明确确认。
- 一次批准只对应一个动作，不能重复使用。
- 本补丁不会绕过 macOS 屏幕录制或辅助功能权限。

## 当前阶段

本批完成的是“可靠性底层 → 实际工作台”的第一轮接线。尚未宣称完整 v2.2.0 完成，后续仍包括项目级 Agent 任务编排、真实文件补丁回滚入口、更多 Skills/工具收据、UI 全盘统一和双架构安装验收。
